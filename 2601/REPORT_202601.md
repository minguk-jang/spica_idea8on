# 1월 아이디어온 보고서

## 경영진 보고용 (요약)

> [!summary] 핵심 요약
> **Everything Claude Code**는 Anthropic의 CLI 기반 AI 코딩 도구인 Claude Code를 실무에서 효율적으로 활용하기 위한 설정 및 워크플로우 가이드입니다. Anthropic x Forum Ventures 해커톤 우승자가 10개월간 실전에서 검증한 설정값을 담고 있으며, Skills/Hooks/Subagents/MCP 등을 유기적으로 연결하여 **반복 작업 자동화** 및 **컨텍스트 효율 극대화**를 실현합니다. 특히 대규모 코드베이스에서 발생하는 컨텍스트 윈도우 부족 문제를 해결하고, 개발 생산성을 높이는 구체적인 방법론을 제시합니다.

## 팀 공유용 (상세 정리)

### 1. Everything Claude Code란?

Claude Code를 단순 보조 도구가 아닌 **'동료 에이전트'**로 만들기 위한 설정 모음집입니다.

| 구성요소 | 설명 |
|---------|------|
| **Skills** | 복잡한 작업을 단축키처럼 정의 (예: `/refactor-clean`, `/tdd`) |
| **Commands** | 즉시 실행 가능한 슬래시 명령어, Skills와 체이닝 가능 |
| **Hooks** | 이벤트 기반 자동화 트리거 (PreToolUse, PostToolUse, Stop 등) |
| **Subagents** | 특정 작업에 특화된 서브에이전트로 작업 위임 |
| **Rules** | 일관된 품질을 위한 불변의 규칙 정의 |
| **MCP** | 외부 서비스(GitHub, Supabase, Vercel 등)와 직접 연결 |

### 2. 핵심 디렉토리 구조

```
~/.claude/
├── skills/       # 워크플로우 정의
├── commands/     # 슬래시 명령어
├── agents/       # 서브에이전트 정의
├── rules/        # AI 규칙 (Best Practices)
└── config.json   # MCP 및 도구 설정
```

### 3. 실전 활용 핵심 원칙

> [!tip] 5가지 핵심 원칙
> 1. **설정은 미세 조정으로 접근** - 과도하게 복잡하게 만들지 말고 필요할 때 조금씩 다듬기
> 2. **컨텍스트 윈도우 관리** - 사용하지 않는 MCP/플러그인은 과감히 비활성화
> 3. **병렬 실행 적극 활용** - `/fork` 명령어나 Git Worktree로 작업 분산
> 4. **반복 작업 자동화** - Hooks를 통해 포맷팅, 린팅 등 자동화
> 5. **서브에이전트 범위 제한** - 명확한 역할과 제한된 도구 부여

### 4. 권장 설정 예시

**Hooks 설정:**
- `PreToolUse`: 장시간 명령어 실행 시 `tmux` 사용 권장, 불필요한 `.md` 파일 생성 차단
- `PostToolUse`: 파일 수정 후 자동 `prettier` 실행, TypeScript 컴파일 체크
- `Stop`: 세션 종료 전 `console.log` 잔존 여부 검사

**MCP 관리:**
- 프로젝트당 활성 MCP는 5~6개로 제한
- 전체 활성 도구(tool) 수는 80개 미만 유지
